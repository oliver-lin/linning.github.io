### 4.1 整体管理
+ 制定初步整体计划，详细分计划，**整体管理方法**合成一致的整体计划；
+ 各项目管理过程组的子过程之间，经常反复多次产生联系；
+ 项目管理整体包括六个过程：
   + 启动 - 制定章程
   + 计划 - 制定项目管理计划
   + 执行 - 指导和管理项目执行
   + 监控 - 监控项目工作、整体变更控制
   + 收尾 - 结束项目或阶段

### 4.2 章程
1. 概念：
   + 授权 项目经理 组织资源；
   + 由实施组织外部签发，签发后，就建立了项目与组织的日常工作之间的联系；
2. 过程：
   + 是制定一份正式批准项目或阶段的文件，并记录反映干系人需要和期望的初步要求的过程
   + 项目经理参与制定
3. 依据
   + 协议，外部客户-合同
   + 项目工作说明书（根据需求、产品或服务要求提供一份工作说明书），外部-招标文件
   + 商业论证，商业角度，是否值得投资
   + 事业环境（组织、公司文化、政府设施）
   + 组织过程资产：（1）组织进行工作的过程和程序；（2）组织整体信息存储检索知识库
5. 启动会议
   + 会议目标
   + 会议准备
   + 通知会议人员
   + 明确会议主题
   + 做好记录
6. 目标
   + 行动对象欲达到的标准。目标应该是具体的、可度量的、可达到的。
7. 引导技术
   + 头脑风暴、会议管理等帮助团队和个人完成关键技术。

### 4.3 制定项目管理计划
1.概念： 
   + 包括：进成质范，风人采沟 计划
   + 确定了执行、监视、控制和结束项目的*方式与方法* 。是其他各个子计划制定的*依据和基础*，它从*整体上指导*项目工作的有序进行。
2. 过程依据
   + 项目章程，作为初始规划的起点。
   + 其他过程的输出结果。都可能导致计划更新。
   + 事业环境因素。（政府、市场、组织、人事、项目管理信息系统）
   + 组织过程资产
3. 工具和技术
   + 专家判断和引导技术

### 5.1 项目范围管理
1.概念
   + 为达到目标，交付特质的产品和服务，所要做的工作。做范围内的事。
   + 三方面工作 
     + 明确项目边界
     + 执行工作进行监督
     + 防止项目范围蔓延 
   + 产品范围（产品要求的描述，是项目范围说明书的重要组成部分）和项目范围（批准的项目范围说明书）
   + 重要性
     + 目标是范围管理的基本依据;
     + 是衡量项目成功的一个必要标准;
     + 使用WBS（工作分解结构）将项目范围分解成可管理的工作包，责任到人; 
   + 范围管理的六个过程
     + 规划：书面描述定义、确认和控制项目范围的过程；
     + 收集需求：记录干系人需求
     + 定义：制定详细过程
     + 创建WBS：可交付成果和工作分解成组件
     + 确认：验收交付成果
     + 控制：<b style="color:red">监控状态</b>，管理范围基准变更过程 
6. 确认范围
   + 正式验收项目*已完成*的*可交付成果*的过程，确保获得客户或发起人的正式验收。
   + 工具和技术是检查和群体决策技术。
     + 步骤：确认的时间；需要哪些投入；被接受的标准；确认会议的组织步骤；确认会议；
     + 检查的问题：可交付成果是否确定；明确里程碑；明确质量标准；审核与承诺是否清晰表达（边界清晰）；覆盖产品或服务的所有活动；风险是否太高；
   + 干系人关注点
     + 管理层，项目进度、资金和资源
     + 客户，可交付成果是否足够完成产品或服务
   + 术语比较
     + 确认范围，可交付成果                 <--> 核实产品，产品是否完成；
     + 确认范围，外部干系人，客户或发起人接受  <--> 质量控制，内部检查，可交付成果正确性;
     + 确认范围，强调核实和接受可交付成果     <--> 收尾，强调产品验收；

## 7。 项目成本管理
1. 概述
   + 构成
     - 项目活动或其组成部分的货币价值或价格。
     - 项目总成本。
   + 重要性
     - 确保再批准的预算内完成项目。生命期成本计算。
     - 成本预测（“财务绩效预测与分析”）
     - 成本估算
   + 成本失控
     - 对项目认识不足
     - 组织制度不健全
     - 方法问题（缺乏科学、严谨的控制方法）
     - 技术制约（项目规划设计不完善、成本计算数据不准确）
     - 需求管理不当（分享失误）
   + 相关术语
     - <b style="color:Chocolate">产品的全生命周期成本：</b>就是在*产品或系统*的整个生命周期内，在获得阶段（设计、生产、安装和测试等活动，即项目存续期间）、运维及生命周期结束时对产品的处置所发生的全部成本。
     + <b style="color:Chocolate">成本类型：</b>
       - 可变成本或变动成本（原材料、燃料）
       - 固定成本（长期借款利息）
       - 直接成本：直接归属项目工作的成本。（团队差旅、工资、项目物料及设备）
       - 间接成本：管理费用科目（保卫费）
       - 机会成本：失去了利用这些资源生产其他最佳替代品的机会
       - 沉没成本：已发生的历史成本（给政府领导送礼，领导被抓了，尚未对项目带来好处，礼白送了。。。）
     + <b style="color:Chocolate">应急和管理储备</b>
       - 应急储备是包含在*成本基准*内的一部分预算。（已知风险 --处理--> 未知工作量）
       - 管理储备不包含在*成本基准*内，但属于总预算和资金需求。（未知风险 --处理--> 未知工作量）
     - <b style="color:Chocolate">成本基准：</b>经批准的按时间安排的**成本支出计划**，并随时反应了经批准的项目**成本变更**，用于**度量**和**监督**项目的实际执行成本。
2. 项目成本管理过程
   + 规划成本：制定政策、程序和文档的过程，为管理提供指南和方向；
     - 管理、花费和控制项目成本而制定政策、程序和文档的过程；作用：提供指南和方向。
     - 输入：项目管理计划；章程；因素；资产 
     - 输出：成本管理计划 
   + 估算成本：完成项目活动资金估算过程；（选择最佳成本方案）
     + 步骤
       - 识别并分析成本的构成科目；
       - 估算每一科目的成本大小；
       - 分析成本估算结果，找出可互相替代的成本，协调比例；  
     + 输入：成本管理计划，人力资源管理计划、范围基准、WBS、WBS字典、进度计划、风险登记册、因素、资产
     + 输出：活动成本估算、估算依据、项目文件更新  
   + 制定预算：单活动或工作包成本估算，建立一个经批准的成本基准的过程；
     - 输入：......、活动成本估算、估算依据、......
     - 输出：成本基准、项目资金需求、项目文件更新
   + 控制成本：监督状态，更新成本、管理变更过程。
     - 输入：项目管理计划、资金需求、工作绩效数据、组织过程资产
     - 输出：工作绩效信息、成本预测、变更请求、项目文件更新、组织过程资产更新 
3. 成本管理技术和工具
   + 分析
      - 技术分析
      - 专家判断
      - 会议
      - 类别估算
      - 参数估算
      - 自下而上（汇总）
      - 三点估算（最可能成本Cm，最乐观成本Co，最悲观成本Cp）
      - 储备分析
      - 质量成本（COQ）
      - 项目管理软件
      - 卖方投标
      - 群体决策
    + 管理
      - 成本汇总（WBS包 --> 控制账户 --> 总成本）
      - 储备分析
      - 历史关系
      - 资金平衡
      + 挣值管理（EVM），监测以下三个关键指标：
        - 计划价值**PV**（计划工作）
        - 挣值**EV**（已完成工作）
        - 实际成本**AC** 
          - SV（schedule 计划、安排），进度偏差 = EV - PV    
            ><b style="color:Gainsboro"> 说明：等于 0 是最好的，表示计划价值全部实现</b>
          - CV（cost 成本） = EV - AC  == BAC（完工预算）- AC
          - SPI（进度绩效指标）= EV/PV    SPI<1.0：未达到计划要求； SPI>1.0： 工作量超过计划；
          - CPI（成本绩效指标）= EV/AC    CPI<1.0：完成工作成本超支；CPI>1.0：成本有结余；
      + 预测
      + 完工尚需绩效指标
      + 绩效审查 

## 8. 项目质量管理 
1. 质量管理基础
   + 质量与项目质量
     +  质量
        + ISO质量：反映**实体**满足**主体***明确和隐含需求*的能力的**特性**总和。
          - 实体是指可单独描述和研究的事物，对象。（活动、产品、过程等）
          - 明确需求：是指标准、规范、合同文件中用户明确提出的要求。 
          - 隐含需求：指用户和社会通过市场调研对实体的期望及不必明确的需求
          - 特性：实体所特有的性质
        + 国家标准质量：一组**固有特性**满足要求的程度 
        - 通俗讲是产品（服务）满足程度的度量，是产品或服务的生命。
        + 质量与等级：质量是作为实现的性能或成果；等级是可交付成果的级别
     + 项目质量
       - 即工作质量。
       - 项目合同是进行项目质量管理的主要依据
   + 质量管理
     + 概念
       - 指确定质量方针、目标和职责，并通过质量体系中的质量规划、保证和控制以及质量改进来使其实现所有管理职能的全部活动。
       - 为实现质量目标而进行的所有质量性质的活动。
     + 方针与目标
       - 方针：组织最高管理者发布的宗旨和方向。组织内部行为准则。
       - 目标：目的，落实方针的具体要求。尽量量化描述，分解落实到项目成员。 
   + 标准体系
     - ISO 9000 == GB/T 19000-2000
     - 8项原则：顾客为中心；领导作用（宗旨和方向）；全员参与；过程方法（过程管理）；管理的系统方法；持续改进；基于事实的决策方法；与供方互利；
     + 全面质量管理（TQM）
       - 全员、全过程、全企业的品质管理。
       - 全员参与为基础，永续经营为目的。
       - 由结构、技术、人员和变革推动者4个要素组成。
       + 4个核心特征：
         - 全员参加的质量管理
         - 全过程的质量管理
         - 全面方法的质量管理（科学的管理方法）
         - 全面结果的质量管理（产品质量、工作质量和服务质量）
     + 6σ（六西格玛）方法
       - 每百万不合格品率少于3.4
       - 核心：DMAIC，所有工作作为一种流程，采用量化方法分析流程中影响质量的因素
       - 项目实施中改进和保证质量 
     + 软件过程改进和能力成熟度模型
       - 软件能力成熟度模型/集成软件能力成熟度模型(CMM/CMMI)，包含了从产品 需求提出、设计、开发、编码、测试、交付运行到产品退役各个过程的各项基本要素。
       - 目标：保证产品或服务质量；时间控制；最低成本
